# Writing Style Matching (Strapi)

## Mandatory Workflow
1. **Analyze first** - Read 3-5 existing pieces in `src/api/`. Identify: sentence length, vocabulary level, tone, recurring phrases, punctuation style, paragraph length, structure patterns, perspective (I/we/you), target audience.
2. **Document voice profile** before writing:
   `Tone: [X] | Sentences: [X] | Vocabulary: [X] | Perspective: [X] | Patterns: [X]`
3. **Write** matching the profile - new content must read like the same author
4. **Verify** - Compare against existing: same rhythm, vocabulary, formatting?

## NEVER
- Write in generic AI tone when existing content has personality
- Switch between formal and casual within one piece
- Use vocabulary the existing content never uses
- Default to "In this article, we will explore..." openings
- Ignore established paragraph/section structure
